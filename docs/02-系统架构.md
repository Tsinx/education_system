# 系统架构

## 总体架构

```text
Browser(React + Antd)
   |
   | HTTP /api/v1
   v
FastAPI (backend/app/main.py)
   |
   +-- MaterialRepository (SQLite)
   +-- AiResultRepository (SQLite)
   +-- MaterialWorker (后台处理线程)
   +-- RAG Service (local / remote)
```

## 前端模块

- `src/layouts/AppLayout.tsx`：整体布局与导航。
- `src/pages/*`：业务页面（课程、图谱、设置等）。
- `src/api/client.ts`：统一 API 调用封装。
- `src/types/*`：接口类型定义。
- `src/data/mock.ts`：后端不可用时的降级演示数据。

## 后端模块

- `app/main.py`：应用入口、CORS、中间件、生命周期。
- `app/api/router.py`：统一挂载 `/api/v1`。
- `app/api/routes/*`：业务路由。
- `app/services/material_repository.py`：核心仓储与知识图谱数据读写。
- `app/services/material_worker.py`：资料处理异步任务。
- `app/services/local_*`：本地向量检索、重排与 RAG。

## 数据流

1. 前端上传资料至 `/api/v1/materials/upload`。
2. 后台处理任务抽取章节、知识点、关系。
3. 前端读取 `/api/v1/knowledge/courses/{course_id}/graph` 绘制图谱。
4. 用户可删除节点并递归或提升子节点。
5. 可导出 `/api/v1/knowledge/courses/{course_id}/graph-export` 供学习通导入。

## 关键设计点

- 路由清晰：按 `projects/courses/materials/generation/knowledge/rag` 分域。
- 可扩展模型层：通过配置切换推理提供方。
- 数据可回放：AI 结果、知识点和关系均入库。
- 前端可降级：接口失败时可退回 mock，保障演示稳定。
