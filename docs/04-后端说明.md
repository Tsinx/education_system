# 后端说明

## 目录

- `app/main.py`：应用入口。
- `app/api/router.py`：统一 API 路由。
- `app/api/routes/`：领域路由。
- `app/services/`：仓储、处理流水线、RAG 相关服务。
- `app/schemas/`：Pydantic 数据模型。

## 关键配置

配置类在 `app/core/config.py`，默认读取 `.env`。

核心配置项：

- `cors_origins`：前端跨域白名单。
- `data_dir`、`material_db_name`：SQLite 数据目录和文件名。
- `llm_provider`：模型供应商选择。
- `rag_*`：检索与重排参数。
- `knowledge_graph_refine_*`：知识图谱关系精炼参数。

## 生命周期

应用启动时：

1. `repository.init()` 初始化数据库结构。
2. `ai_result_repo.init()` 初始化 AI 结果库。
3. `worker.start()` 启动后台处理线程。

应用关闭时：

1. `worker.stop()` 停止后台线程。

## 知识图谱相关能力

- 查询课程图谱：`GET /api/v1/knowledge/courses/{course_id}/graph`
- 删除节点：`DELETE /api/v1/knowledge/courses/{course_id}/nodes/{node_id}`
- 导出学习通模板：`GET /api/v1/knowledge/courses/{course_id}/graph-export`

删除节点支持两种策略：

- 递归删除子节点。
- 不递归删除时，将子节点提升一级。

## 开发建议

- 新增接口时优先在 `schemas` 中定义响应模型。
- 仓储层统一处理 SQLite 语句，路由层只做参数校验和组合。
- 对长耗时任务走 worker 流程，避免阻塞请求线程。
