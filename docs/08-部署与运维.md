# 部署与运维

## 本地部署（开发环境）

1. 启动后端：`uvicorn app.main:app --reload --port 8000`
2. 启动前端：`npm run dev`
3. 访问前端：`http://localhost:5173`
4. 访问后端文档：`http://localhost:8000/docs`

## 配置管理

后端从 `.env` 读取配置，关键包括：

- 模型供应商与 API Key
- RAG 检索参数
- 数据目录和数据库文件

建议：

- 将 `.env.example` 作为团队统一模板。
- 生产环境通过环境变量注入，不直接写入仓库。

## 日志与诊断

建议关注：

- 后端启动日志
- 资料处理任务状态
- 导出接口状态码与耗时

常见诊断方式：

- `GET /api/v1/health` 检查服务可用性
- 访问 `/docs` 验证路由加载是否完整

## 数据备份

当前主数据落于 SQLite：

- 课程/资料/知识点/关系
- AI 结果

建议：

1. 每日备份数据库文件。
2. 重要版本导出知识图谱与 AI 结果产物。
3. 升级前先做数据库快照。

## 发布建议

- 先执行前端构建与后端基础语法检查。
- 发布前至少回归：资料上传、图谱查看、图谱删除、图谱导出、生成任务。
- 对外演示前，准备 mock 兜底数据和稳定网络策略。
