# 习题生成功能更新方案

## 1. 当前项目现状

- 后端已有统一生成链路：`/api/v1/generation/start` 创建任务，`/api/v1/generation/stream/{result_id}` 通过 SSE 流式回传内容，`ai_results` 表负责结果入库。
- 前端已有统一 AI 操作台：课程详情页 [CourseDetailPage.tsx](D:\Projects\education_system\frontend\src\pages\CourseDetailPage.tsx) 负责发起生成、查看任务状态、实时预览与导出。
- 知识点数据已经入库且可树状组织：`/api/v1/knowledge/courses/{course_id}/graph` 返回节点、层级、父子关系，可直接复用为“知识点复选树”数据源。
- 现有生成类型包括课程大纲、知识库、教学计划、思政案例、教案；尚未落地“习题生成”。
- 现有 agentic 能力主要体现在“前置规划 + 正式生成”两段式提示链路；尚未实现“LLM 请求 Python 执行并根据结果迭代修正”的闭环。

## 2. 本次目标

- 新增“习题生成”产物类型，纳入现有统一任务系统。
- 在生成时支持：
  - 输入教师要求；
  - 复选知识点；
  - 若课程已有知识点，则以前端树状展开方式选择。
- 后端采用 agentic 流程：
  1. 先分析题目要求与选中知识点的易错点；
  2. 再生成题目；
  3. 生成参考答案与解析；
  4. 对选择题单独生成干扰项，并在服务端打乱选项；
  5. 对计算题通过“LLM 请求执行 Python → 服务端运行 → 把结果反馈给 LLM → LLM 继续修正/确认”的方式完成校验。

## 3. 更新范围

## 3.1 后端

- 扩展 `AiOutputType`，增加 `exercise`。
- 扩展 `AiGenerateRequest`，支持习题生成专属参数：
  - `selected_knowledge_ids`
  - `exercise_requirements`
- 在 `generation.start` 中为 `exercise` 创建任务上下文。
- 在 `generation.stream` 中为 `exercise` 走专门的 agentic 习题生成链路，而不是复用通用文稿生成模板。
- 新增习题生成服务，负责：
  - 聚合课程信息与选中知识点；
  - 生成易错点分析；
  - 生成结构化题目草案；
  - 选择题干扰项打乱与答案映射；
  - 计算题 Python 执行循环；
  - 将结构化结果渲染为 Markdown。

## 3.2 前端

- 在课程详情页 AI 操作台新增“习题生成”按钮。
- 生成弹窗中，当选择“习题生成”时展示：
  - 习题要求输入框；
  - 课程知识点复选树；
  - 已选知识点数量提示。
- 继续复用已有任务队列、SSE 预览、结果预览与导出能力。
- 调整任务阶段展示逻辑，使 exercise 的 agentic 状态能显示更细的阶段信息。

## 3.3 文档

- 更新 API 清单，补充习题生成请求参数说明。
- 更新前端说明，记录“知识点树选择 + 习题生成”交互。
- 方案文档保留当前实现思路，便于后续继续扩展成题库、批量出卷与难度分层。

## 4. 详细实现设计

## 4.1 习题生成请求模型

- 仍复用 `POST /generation/start`。
- 当 `output_types` 中包含 `exercise` 时：
  - `user_guidance` 继续保留，兼容统一入口；
  - `exercise_requirements` 作为更明确的习题要求字段；
  - `selected_knowledge_ids` 记录用户勾选的知识点节点。
- 服务端会将选中节点解析为知识点名称和层级链路，写入 `request_context`，供流式阶段与结果回放使用。

## 4.2 Agentic 习题生成链路

### 阶段 A：知识与易错点分析

- 输入：
  - 课程名称、课程简介；
  - 选中的知识点树（若未选则回退到整门课知识库）；
  - 教师要求。
- 目标：
  - 分析本次命题范围；
  - 识别高频易错点、混淆点、计算陷阱；
  - 形成结构化命题依据。

### 阶段 B：题目草案生成

- LLM 输出结构化 JSON，而不是直接输出 Markdown。
- 题目至少区分：
  - `single_choice`
  - `multiple_choice`
  - `short_answer`
  - `calculation`
- 每题包含：
  - 题干；
  - 关联知识点；
  - 设计意图/对应易错点；
  - 参考答案；
  - 解析。

### 阶段 C：选择题干扰项与打乱

- 要求 LLM 输出：
  - 正确答案内容；
  - 若干干扰项内容；
  - 每个干扰项对应的误区说明。
- 由服务端统一打乱选项顺序，重新映射答案字母，避免固定把正确答案放在某个位置。
- Markdown 输出时展示：
  - 题目；
  - 打乱后的 A/B/C/D；
  - 参考答案；
  - 解析；
  - 可选地附带“干扰项命题依据”。

## 4.3 计算题 Python 校验闭环

- 采用显式动作循环，不依赖模型原生工具调用协议。
- 服务端给 LLM 一个严格 JSON 协议，允许它返回两类动作：
  - `python`
  - `final`
- `python` 动作包含：
  - `reason`
  - `code`
- 服务端执行该 Python 代码，并把：
  - 退出码；
  - 标准输出；
  - 标准错误
  回传给 LLM。
- LLM 可基于执行结果继续：
  - 修正代码；
  - 修正中间推导；
  - 或给出最终答案。
- 终止条件：
  - LLM 返回 `final`；
  - 或达到最大迭代次数并使用最后一次成功结果兜底。
- 这样可以显式满足“LLM 调用 Python 解释器并运行，直到它认为结果满意”的要求。

## 4.4 结果渲染格式

- 最终仍写入 Markdown，兼容现有预览与导出。
- 推荐结构：
  - 习题生成说明
  - 易错点分析
  - 习题列表
  - 参考答案与解析
  - 计算题 Python 校验记录摘要
- 这样既保留可读性，也保留 agentic 过程痕迹。

## 5. 执行计划

1. 扩展类型、请求模型与结果标题映射。
2. 新增后端习题生成服务，实现：
   - 易错点分析；
   - 结构化出题；
   - 选择题打乱；
   - 计算题 Python 循环校验；
   - Markdown 渲染。
3. 接入 `generation.stream`，为 `exercise` 走专用流式链路。
4. 前端课程详情页新增“习题生成”按钮与配置 UI。
5. 接入知识点树复选与参数提交。
6. 更新任务阶段展示、文档与基础验证。

## 6. 风险与处理

- 风险：LLM 输出 JSON 不稳定。
  - 处理：复用现有 JSON 解析兜底，并在服务端做字段归一化。
- 风险：计算题 Python 代码执行失败。
  - 处理：引导 LLM 看到 stderr 后重试，且限制最大迭代轮数。
- 风险：知识点树节点较多时弹窗拥挤。
  - 处理：仅在选择“习题生成”时加载，并使用 Ant Design 的 `Tree`/`TreeSelect` 风格折叠展示。
- 风险：选择题答案顺序规律化。
  - 处理：服务端统一打乱，永不直接使用模型原始顺序。

## 7. 本轮验收点

- 可以在课程详情页发起“习题生成”。
- 可以输入要求并复选知识点。
- 后端能输出包含“易错点分析 + 题目 + 答案解析”的结果。
- 选择题选项由服务端打乱。
- 计算题会经过 Python 执行校验链路。
