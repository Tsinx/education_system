# 教师课程 AI 智能体平台项目设计文档

## 1. 项目背景与目标

本项目面向学校教师在教学准备阶段的高频需求，支持教师上传课程资料后，由 AI 智能体自动生成：

- 课程大纲
- 课程知识库（知识点结构化）
- 课堂设计（含教学活动流程）
- 教案
- 课堂练习题
- 课件大纲与 PPT 生成任务描述

项目采用前后端分离架构，支持后续扩展多模型、多学校、多学科场景。

## 2. 角色与核心业务流程

### 2.1 用户角色

- 教师：上传资料、发起生成任务、查看编辑结果、导出内容
- 管理员（预留）：管理模型配置、系统提示词模板、用户权限

### 2.2 主流程

1. 教师创建课程项目
2. 上传课程资料（PDF、Word、PPT、图片、文本）
3. 系统进行解析、清洗、分段、向量化，构建课程知识库
4. 教师选择生成内容类型并设置参数（学段、课时、难度、风格）
5. AI 智能体执行编排链路并输出结构化内容
6. 教师在线预览、二次编辑、导出（Markdown/Word/PPT 任务包）

## 3. 技术选型与理由

### 3.1 前端

- 技术栈：Vite + React + TypeScript + Ant Design + React Router
- 选择理由：
  - Vite 启动快、构建快，适合竞赛阶段快速迭代
  - React 生态成熟，组件化利于复杂界面搭建
  - TypeScript 提升项目可维护性，利于后期协作
  - Ant Design 提供高质量表单、表格、步骤条、抽屉等企业级组件，能快速完成可视化系统

### 3.2 后端

- 技术栈：Python + FastAPI + Pydantic + Uvicorn
- 选择理由：
  - FastAPI 对异步请求、类型校验、接口文档支持优秀
  - Python 便于接入 AI 模型、向量数据库、文档解析链路
  - 天然生成 OpenAPI 文档，便于前后端联调

### 3.3 数据与 AI（阶段规划）

- 结构化数据：PostgreSQL
- 向量检索：Milvus / pgvector（可替换）
- 任务队列：Celery + Redis（大文件和长任务）
- LLM 接入：统一适配层（支持多模型供应商）

## 4. 系统架构设计

### 4.1 架构分层

- 前端应用层：课程管理、知识库管理、内容生成工作台、结果编辑与导出
- API 网关层：鉴权、限流、请求编排
- 业务服务层：
  - 资料处理服务（解析、切片、清洗）
  - 知识库服务（向量索引、检索）
  - 智能体编排服务（大纲/教案/习题/PPT 生成）
  - 导出服务（文档与课件）
- 数据层：关系数据库 + 向量数据库 + 对象存储

### 4.2 智能体能力拆分

- OutlineAgent：生成课程大纲
- KnowledgeAgent：提取并组织知识点图谱
- LessonPlanAgent：生成教案与课堂流程
- ExerciseAgent：生成练习题与答案解析
- PPTAgent：输出可用于 PPT 自动生成的结构化脚本

## 5. 模块设计

### 5.1 前端模块

- 仪表盘：查看项目状态、任务进度
- 课程项目：创建课程、上传资料、管理版本
- 生成中心：配置生成参数并触发任务
- 结果中心：预览、编辑、导出生成内容

### 5.2 后端模块

- `/api/v1/projects`：课程项目管理
- `/api/v1/materials`：资料上传与解析任务
- `/api/v1/generation`：生成任务创建与查询
- `/api/v1/knowledge`：知识库查询与可视化
- `/api/v1/health`：健康检查

## 6. 数据对象（核心）

- Project：课程项目
- Material：上传资料
- GenerationTask：生成任务
- Artifact：生成产物（大纲/教案/习题/PPT脚本）
- KnowledgeChunk：知识片段

## 7. 非功能需求

- 安全：JWT 鉴权、文件类型白名单、上传大小限制
- 性能：异步任务处理、分页与缓存
- 可观测：统一日志、请求追踪、错误告警
- 可扩展：Agent 接口标准化，支持新增教学产物类型

## 8. 竞赛版本实施范围（MVP）

- 支持教师创建课程与上传资料（当前为接口骨架与模拟数据）
- 支持选择模板一键生成 5 类教学产物（模拟生成）
- 前端完成完整流程界面与任务状态展示
- 后端提供统一 API 契约，便于后续接入真实模型和数据库

## 9. 里程碑建议

- M1：前后端框架搭建与页面流程打通
- M2：文档解析与知识库构建
- M3：多智能体编排与结果质量优化
- M4：导出能力与演示优化
